# Etapa 1 : Fundamentos de Git

# Descripcion Etapa

Este repositorio fue creado commo parte de mi formacion profesional, con el fin de aplicar los Fundamentos de Git y github, utilizando buenas practicas de control de versiones las cuales son utilizadas dia a dia en entornos reales de desarrollo del software.

# Objetivo del Proyecto

-Comprender el flujo basico de Git (add,commit,push)
-Comprender la diferencia entre repositorio local y remoto
-Usar ramas para generar entornos de trabajo seguros
-Gestionar un repositorio remoto en GitHub
-Comprender como deshacer cambios de forma segura (restore,revert)
-Mantener un historial de cambios claro y ordenado

# Flujo de trabajo utilizado

Cambio --> Staging --> Commit --> Push

Se realizaron Ejercicios Practicos para Garantizar:

-Control total de los cambios realizados
-Realizacion de commits con mensajes descriptivos
-Historial entendible para otros Desarrolladores

# Uso de Branches

Se realizaron Ejercicios Practicos con el fin de:

-Desarrollar cambios fuera de main
-Proteger la rama principal de fallos
-Integrar cambios mediante merge

# Deshacer Cambios en Git

Se practicaron los siguientes escenarios reales

-Descartar cambios no agregados al staging (git restore nombreArchivo)
-Quitar Archivos del staging (git restore --staged nombreArchivo)
-Revertir commits de forma segura sin borrar el historial (git revert HEAD)

Estas practicas de escenarios reales nos permite saber que hacer en caso de que nos equivoquemos y poder resolverlo de manera profesional
